# Mindestanforderungen & Erweiterungen

- [ ] Funktionierender Login mit Master-Passwort.
- [ ] Sicheres Hashing des Master-Passworts (Bcrypt/Argon2).
- [ ] Speichern von Passwörtern mit Verschlüsselung in der Datenbank.
- [ ] CRUD: Anzeigen und Hinzufügen von Passwörtern funktioniert.
- [ ] Grundlegendes Sicherheitskonzept im README dokumentiert.

## Erweiterungen (für Note 5.0 - 6.0)
- [ ] **Passwort-Stärke-Anzeige:** Visuelles Feedback während der Eingabe.
- [ ] **Passwort-Generator:** Einstellbare Parameter (Länge, Sonderzeichen).
- [ ] **Erweiterte Security Features:** Automatisches Ausblenden von Passwörtern nach 10 Sekunden.
- [ ] **Vollständiges Journal:** Block-Dokumentation mit Problemstellungen im README.
- [ ] **Search/Filter:** Effizientes Suchen in der Passwortliste.
- [ ] **Input Validation:** Strenge Validierung auf Server- und Clientseite gegen Injection.
- [ ] **Security Logging:** Implementierung eines Log-Systems, das keine PII (Personally Identifiable Information) oder Secrets enthält.
